YT ULTIMATE CLI - FUTURE UPDATE / RECOMPILE GUIDE
====================================================

Project:
  YT Ultimate CLI
  Main command:
    ytdown
  Windows EXE:
    ytdown.exe
  GitHub repo:
    https://github.com/Ibrahim71Reza/ytultimate.git

Purpose of this note:
  Follow this file when official yt-dlp, FFmpeg, Deno, PyInstaller, Python dependencies,
  or your own YT Ultimate CLI source code need updating in the future.

IMPORTANT TRUTH
===============

1. Windows ytdown.exe does NOT auto-update itself.
   If yt-dlp updates, you must rebuild ytdown.exe and upload a new GitHub Release.

2. Linux/source installs can update yt-dlp more easily, but the app source still needs:
     git pull
     pip install -e .

3. FFmpeg is external.
   Your tool uses system FFmpeg/ffprobe. It is not bundled inside the current ytdown.exe.

4. Deno is external.
   It helps yt-dlp / YouTube EJS challenge solving when used with:
     --remote-components ejs:github

5. For YouTube reliability, recommended command:
     ytdown download "VIDEO_URL" --quality 1080p --fallback nearest --remote-components ejs:github


WINDOWS FULL UPDATE + EXE REBUILD GUIDE
=======================================

Default Windows project path:
  D:\ytultimate

Default Windows output directory used by the tool:
  C:\Users\<YOUR_USERNAME>\Videos\YTUltimate

Default Windows config path:
  C:\Users\<YOUR_USERNAME>\AppData\Local\ytultimate\ytultimate\config.json

Default Windows queue path:
  C:\Users\<YOUR_USERNAME>\AppData\Local\ytultimate\ytultimate\queue.jsonl


STEP 1 - Open PowerShell
------------------------

Open PowerShell and go to project folder:

  cd D:\ytultimate


STEP 2 - Confirm Git is clean
-----------------------------

  git status -sb

Expected clean output:

  ## main...origin/main

If there are uncommitted changes, commit them or restore them before rebuilding.


STEP 3 - Pull latest source from GitHub
--------------------------------------

  git pull


STEP 4 - Fix temporary folder variables if needed
------------------------------------------------

Use this if PowerShell/Git shows temp warnings:

  $env:TEMP="$env:LOCALAPPDATA\Temp"; $env:TMP="$env:LOCALAPPDATA\Temp"


STEP 5 - Update FFmpeg on Windows
---------------------------------

If FFmpeg is already installed:

  winget upgrade Gyan.FFmpeg

If not installed:

  winget install Gyan.FFmpeg

Close and reopen PowerShell after install/upgrade.

Check:

  ffmpeg -version
  ffprobe -version


STEP 6 - Update Deno on Windows
-------------------------------

If Deno is already installed:

  winget upgrade DenoLand.Deno

If not installed:

  winget install DenoLand.Deno

Check:

  deno --version


STEP 7 - Clean old build artifacts
----------------------------------

Run from D:\ytultimate:

  Remove-Item -Recurse -Force build, dist, .tmp, .pyinstaller-cache, ytdown.spec -ErrorAction SilentlyContinue


STEP 8 - Create or refresh D-drive virtual environment
-----------------------------------------------------

This keeps the Python environment on D drive instead of filling C drive.

If you want a completely fresh venv:

  Remove-Item -Recurse -Force .venv -ErrorAction SilentlyContinue
  python -m venv D:\ytultimate\.venv

Upgrade build tools:

  .\.venv\Scripts\python.exe -m pip install --upgrade pip setuptools wheel


STEP 9 - Install / update project dependencies
---------------------------------------------

Install the project in editable mode and update key build/runtime packages:

  .\.venv\Scripts\python.exe -m pip install -e .
  .\.venv\Scripts\python.exe -m pip install -U yt-dlp pyinstaller pyinstaller-hooks-contrib rich typer platformdirs

Optional test dependency:

  .\.venv\Scripts\python.exe -m pip install -U pytest


STEP 10 - Check installed versions
----------------------------------

  .\.venv\Scripts\python.exe -m pip show yt-dlp
  .\.venv\Scripts\python.exe -m pip show pyinstaller
  .\.venv\Scripts\python.exe -m pip show ytultimate


STEP 11 - Create EXE launcher file
----------------------------------

Run this one-line PowerShell command:

  New-Item -ItemType Directory -Force build | Out-Null; Set-Content -Encoding UTF8 -Path build\ytdown_exe_entry.py -Value @('from ytu.cli import app','','if __name__ == "__main__":','    app()')


STEP 12 - Compile and test source before EXE build
--------------------------------------------------

  .\.venv\Scripts\python.exe -m compileall ytu

If pytest is installed:

  .\.venv\Scripts\python.exe -m pytest -q

Expected:

  all tests passed


STEP 13 - Test source CLI before EXE build
-----------------------------------------

  .\.venv\Scripts\Activate.ps1

  ytdown --version
  ytdown doctor
  ytdown formats "https://www.youtube.com/shorts/dndq9l2pL7Q" --remote-components ejs:github
  ytdown download "https://www.youtube.com/shorts/dndq9l2pL7Q" --quality 1080p --fallback nearest --remote-components ejs:github --output D:\ytultimate\test-source-download

Expected:
  - doctor should show ffmpeg found and ffprobe found
  - formats should show available formats
  - download should finish with Done.
  - no serious EJS / n challenge warning when --remote-components ejs:github is used


STEP 14 - Build Windows EXE
---------------------------

Run this full one-line command:

  $env:PYINSTALLER_CONFIG_DIR="D:\ytultimate\.pyinstaller-cache"; $env:TEMP="D:\ytultimate\.tmp"; $env:TMP="D:\ytultimate\.tmp"; New-Item -ItemType Directory -Force .tmp,.pyinstaller-cache,build,dist | Out-Null; .\.venv\Scripts\python.exe -m PyInstaller --onefile --console --clean --name ytdown --workpath D:\ytultimate\build --distpath D:\ytultimate\dist --specpath D:\ytultimate --collect-submodules ytu --collect-submodules yt_dlp --collect-submodules typer --collect-submodules rich --hidden-import typer --hidden-import rich --hidden-import yt_dlp build\ytdown_exe_entry.py

Expected EXE:

  D:\ytultimate\dist\ytdown.exe


STEP 15 - Test rebuilt EXE
--------------------------

  .\dist\ytdown.exe --version
  .\dist\ytdown.exe doctor
  .\dist\ytdown.exe formats "https://www.youtube.com/shorts/dndq9l2pL7Q" --remote-components ejs:github
  .\dist\ytdown.exe download "https://www.youtube.com/shorts/dndq9l2pL7Q" --quality 1080p --fallback nearest --remote-components ejs:github --output D:\ytultimate\test-exe-download
  .\dist\ytdown.exe audio "https://www.youtube.com/shorts/dndq9l2pL7Q" --audio-format mp3 --remote-components ejs:github --output D:\ytultimate\test-exe-audio

Expected:
  - version shows ytultimate version
  - doctor shows ffmpeg found and ffprobe found
  - video download works
  - audio extraction works
  - command ends with Done.


STEP 16 - Clean test/build cache before Git commit
--------------------------------------------------

Do NOT delete dist\ytdown.exe if you still need to upload it to GitHub Release.

Safe cleanup:

  Remove-Item -Recurse -Force .tmp, .pyinstaller-cache, test-source-download, test-exe-download, test-exe-audio -ErrorAction SilentlyContinue
  Remove-Item -Force ytdown.spec -ErrorAction SilentlyContinue


STEP 17 - Do NOT commit generated files
---------------------------------------

Never commit these to Git:

  build/
  dist/
  .tmp/
  .pyinstaller-cache/
  *.spec
  test-audio/
  test-downloads/
  test-ejs*/
  test-source-download/
  test-exe-download/
  test-exe-audio/

They should be inside .gitignore.


STEP 18 - Commit source changes only
------------------------------------

Check:

  git status

Add only real source/docs/config changes, for example:

  git add ytu README.md CHANGELOG.md config.example.json .gitignore scripts
  git commit -m "update yt-dlp compatibility and rebuild release"
  git push

If only dependency refresh happened and source code did not change, you may not need a commit.
But you still need a new GitHub Release if you rebuilt ytdown.exe.


STEP 19 - Create GitHub Release
-------------------------------

Go to:

  GitHub repo -> Releases -> Draft a new release

Use a new tag, for example:

  v1.0.3

Release title example:

  YT Ultimate CLI v1.0.3 - Windows EXE Refresh

Upload:

  D:\ytultimate\dist\ytdown.exe

Do not upload the whole dist folder. Upload only ytdown.exe.

VERSIONING RULE
===============

If only ytdown.exe was rebuilt with a newer yt-dlp but your app code did not change:
  Use patch version, for example:
    v1.0.3

If app features changed:
  Use patch or minor version depending on change size:
    v1.0.3
    v1.1.0

If breaking changes happen:
  Use major version:
    v2.0.0

TROUBLESHOOTING
===============

Problem:
  ModuleNotFoundError inside ytdown.exe

Cause:
  PyInstaller built using the wrong Python environment.

Fix:
  Always build with:
    .\.venv\Scripts\python.exe -m PyInstaller ...

And confirm log says:
    Python environment: D:\ytultimate\.venv


Problem:
  ffmpeg missing / ffprobe missing

Fix:
  Windows:
    winget install Gyan.FFmpeg

  Linux:
    sudo apt install ffmpeg

Then reopen terminal and run:
    ytdown doctor


Problem:
  YouTube EJS / signature / n challenge warning

Fix:
  Install Deno and use:
    --remote-components ejs:github

Example:
    ytdown download "VIDEO_URL" --quality 1080p --fallback nearest --remote-components ejs:github


Problem:
  Git shows build/test files

Fix:
  Make sure .gitignore contains:
    build/
    dist/
    .tmp/
    .pyinstaller-cache/
    *.spec
    test-ejs*/
    test-downloads/
    test-audio/

Then remove generated files:
  git clean -fdX

Careful:
  git clean -fdX deletes ignored files. Do not use it if you still need dist/ytdown.exe before uploading.
